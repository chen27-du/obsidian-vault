# 「AI深度思考工具」产品化全方案

---

## 一、产品定位

### 一句话定义

> 一套**引导用户用AI对任何一个观点进行深度榨取**的结构化思考工具——用户输入一个观点，工具自动生成六个阶段的提问流程，引导用户与AI完成从发散到元认知的完整思考闭环。

### 核心价值主张

| 用户痛点 | 产品解法 |
|----------|----------|
| 用AI只会问一句答一句，得不到深度答案 | 预设六阶段结构化提问流程，自动引导深度对话 |
| 不知道该怎么追问，问两层就断了 | 内置五层追问法，每层自动生成下一步追问 |
| 想法碎片化，无法形成体系 | 第三阶段自动将碎片结构化为推理链条 |
| 找不到自己论证的漏洞 | 内置苏格拉底诘问+钢铁人论证，自动对抗 |
| 学了很多方法论但不知道怎么落地 | 第五阶段自动适配个人场景生成执行方案 |
| 不知道自己用AI的方式对不对 | 第六阶段自动复盘对话质量 |

### 产品名建议（供参考）

- **深思 DeepThink** —— 直接传达核心功能
- **榨取 SqueezeAI** —— 延续原文"压榨AI潜力"的概念
- **思维钻探 MindDrill** —— 呼应"认知钻探"的方法论
- **六层 SixLayers** —— 对应六个阶段

## 二、目标用户

### 核心用户画像

| 用户群体 | 典型场景 | 付费意愿 |
|----------|----------|----------|
| **知识工作者** | 写深度文章前梳理观点、做决策前多角度分析 | 高（效率工具） |
| **创业者/管理者** | 验证商业想法、找论证漏洞、做战略推演 | 高（决策质量） |
| **内容创作者** | 选题深度挖掘、构建独特观点体系 | 中高（内容差异化） |
| **大学生/研究生** | 论文选题论证、学术观点打磨 | 中（教育场景） |
| **终身学习者** | 把碎片知识体系化、检验自己的认知盲区 | 中（自我提升） |

### 优先级

**MVP阶段聚焦第一类：知识工作者和内容创作者**——他们有高频需求、有付费习惯、且产品效果最容易被感知（"我用这个工具写出了一篇深度文章"）。

## 四、产品功能架构

### 4.1 核心功能流程

```
用户输入一个观点/问题
        ↓
系统自动识别主题类别
（教育/科技/商业/人生/社会/健康）
        ↓
自动匹配12角色组合
（用户可手动替换任意角色）
        ↓
【第一阶段】多角色视角扫描 → AI生成12个视角
    用户标记"意外"角度
        ↓
【第一阶段】反向压力测试 → AI生成8+反驳论点
    用户逐条标注"成立/不成立"
        ↓
【第一阶段】边界条件探测 → AI生成适用边界
        ↓
【第二阶段】五层追问法 → 逐层引导深挖
    用户选择最有价值的角度
    系统自动判断"重复/推进"并调整追问
        ↓
【第三阶段】第一性原理重构 → 自动拆解并重建推理链
        ↓
【第三阶段】多模型交叉验证 → 5个模型独立分析+收敛检测
        ↓
【第四阶段】苏格拉底诘问 → AI连续追问，用户作答
        ↓
【第四阶段】钢铁人论证 → 构建最强反方论证
        ↓
【第五阶段】极端案例测试 → 3个真实案例验证
        ↓
【第五阶段】个人适配 → 输入个人画像，生成执行方案
        ↓
【第六阶段】过程复盘 → 自动分析提问质量+优化建议
        ↓
输出完整思考报告（可导出PDF/Markdown）
```

### 4.2 功能模块拆解

#### 模块一：观点输入与主题识别

| 功能 | 说明 |
|------|------|
| 观点输入框 | 用户输入任意观点/问题，支持中英文 |
| 主题自动分类 | 基于NLP识别主题类别，推荐对应角色组 |
| 角色自定义 | 用户可替换系统推荐的角色，支持自定义角色 |
| 历史观点库 | 保存用户历史分析过的观点，支持回顾和对比 |

#### 模块二：六阶段引导引擎（核心）

| 功能 | 说明 |
|------|------|
| 阶段导航 | 左侧显示六个阶段的进度条，当前阶段高亮 |
| 提示词自动生成 | 根据用户输入的观点，自动填充每个步骤的提示词模板 |
| AI对话窗口 | 每个步骤对应一个独立的对话窗口，避免上下文混乱 |
| 用户标注系统 | 在关键节点提供标注工具（"意外""成立""不成立""盲区"等标签） |
| 阶段间信息传递 | 前一阶段的标注和选择自动传递到后一阶段作为输入 |

#### 模块三：五层追问引擎

| 功能 | 说明 |
|------|------|
| 角度选择器 | 用户从第一阶段的输出中选择最有价值的角度 |
| 逐层追问 | 系统自动基于上一层回答生成下一层追问 |
| 重复检测 | 若AI回答与上一层重复，系统自动提示并建议换问法 |
| 因果链可视化 | 五层回答自动串联为因果链图 |

#### 模块四：结构化输出引擎

| 功能 | 说明 |
|------|------|
| 第一性原理拆解器 | 自动将论证拆解为基本事实→逻辑推导→结论 |
| 多模型分析器 | 5个模型并行分析，自动检测结论收敛/发散 |
| 论证结构图 | 自动生成论证的逻辑结构可视化 |

#### 模块五：对抗与修正引擎

| 功能 | 说明 |
|------|------|
| 苏格拉底模式 | AI只问不答，用户在对话框中作答 |
| 盲区标记 | 用户回答不上的问题自动标记为"知识盲区" |
| 钢铁人构建器 | 自动构建最强反方论证 |
| 修正追踪 | 记录用户在对抗过程中对原始观点的每一次修正 |

#### 模块六：个人适配方案生成器

| 功能 | 说明 |
|------|------|
| 个人画像输入 | 用户填写年龄/职业/现状/约束条件 |
| 方案生成 | 基于前面所有分析生成个性化执行方案 |
| 行动计划 | 自动生成每周/每日计划，支持导出到日历 |
| 里程碑追踪 | 设定里程碑并支持定期check-in |

#### 模块七：报告与导出

| 功能 | 说明 |
|------|------|
| 思考报告 | 将六个阶段的全部输入和输出整合为一份结构化报告 |
| 导出格式 | PDF / Markdown / 微信公众号排版格式 |
| 分享功能 | 生成可分享的思考报告链接（脱敏后） |
| 对比功能 | 对比同一观点在不同时间的分析结果 |

### 5.2 关键技术决策

| 决策点 | 建议方案 | 原因 |
|--------|----------|------|
| AI模型 | 主用Claude 3.5/GPT-4o，备用国产模型（DeepSeek/Qwen） | 深度思考需要强推理能力；国产模型做降级和成本优化 |
| 提示词管理 | 提示词模板化+动态变量注入 | 11个步骤的提示词需要统一管理和版本控制 |
| 上下文管理 | 每个阶段独立会话+跨阶段摘要传递 | 避免上下文过长导致质量下降 |
| 流式输出 | SSE（Server-Sent Events） | AI生成内容较长，需要实时展示 |
| 多模型并行 | Step 6（多模型交叉验证）需要同时调用5个不同视角的prompt | 用队列+并发控制 |

### 5.3 提示词工程（核心壁垒）

这是产品的**核心竞争力**——不是简单的"接一个API"，而是：

1. **11套精心设计的提示词模板**，每套包含：
   - 系统角色设定
   - 输出格式约束
   - 质量控制指令（如"不要给出通用答案""每个观点必须有具体例证"）
   - 动态变量（用户输入的观点、前序阶段的输出等）

2. **阶段间的"信息压缩"逻辑**：前一阶段可能产生大量文本，需要压缩为结构化摘要传递给下一阶段

3. **"重复检测"算法**：在五层追问中，自动判断AI是否在重复上一层的内容

## 七、MVP开发计划

### 7.1 MVP范围（只做最小可用版本）

**砍掉的：**
- 团队版/协作功能
- 多模型并行（Step 6先用单模型模拟）
- 复杂的可视化（因果链图等）
- 小程序端
- 教育版

**保留的：**
- 观点输入 + 主题分类
- 六阶段顺序引导（每阶段一个对话窗口）
- 基础标注系统（"意外""成立""不成立"三个标签）
- 五层追问的逐层引导
- 基础报告导出（Markdown格式）
- 免费版（3次/月）+ 专业版付费

### 7.2 开发排期

| 周次 | 任务 | 产出 |
|------|------|------|
| **第1周** | 产品原型设计（Figma）+ 提示词模板v1编写 | 可交互原型 + 11套提示词初稿 |
| **第2-3周** | 前端开发：观点输入+阶段导航+对话窗口 | 可运行的前端页面 |
| **第3-4周** | 后端开发：提示词编排引擎+AI调用层+流式输出 | API接口 |
| **第5周** | 前后端联调+标注系统+报告导出 | 可端到端跑通的MVP |
| **第6周** | 提示词调优（用10个不同观点测试每个阶段的输出质量） | 优化后的提示词模板 |
| **第7周** | 用户测试（邀请20人试用）+ 收集反馈 | 反馈报告 |
| **第8周** | Bug修复+付费系统接入+上线 | 上线版本 |

**总计：8周，1-2人团队可完成。**

## 八、风险与应对

| 风险 | 概率 | 应对 |
|------|------|------|
| AI输出质量不稳定 | 高 | 提示词模板中加入质量控制指令；用户反馈机制；定期调优 |
| 用户嫌流程太长，中途放弃 | 高 | 支持"跳过某个阶段"；支持"只跑某几个阶段"；进度保存 |
| AI API成本过高 | 中 | 设置免费版次数上限；缓存相同观点的分析结果；用国产模型降成本 |
| 大厂抄一个类似功能 | 中 | 速度+社区+方法论品牌是壁垒；大厂做的是通用功能，很难做到这种深度 |
| 用户觉得"我自己也能在ChatGPT里做" | 中 | 产品价值在于**结构化引导+跨阶段信息传递+标注系统+报告输出**——这些是纯ChatGPT做不到的 |

```
你是一个深度分析助手。现在请你完成以下任务：

## 任务
```
